## Affected Commands

- `brandkit build <campaign> --offline` - deterministic single-campaign path.
- `brandkit build <campaign>` - online copy/background generation and explicit provider failures.
- `brandkit build-all --offline` - offline flag propagation and aggregate failure status.
- `brandkit verify` - campaign-scoped visual/content verification.
- `brandkit validate` - campaign-scoped channel validation and diff result.
- Clean-clone install and first build on Ubuntu 24.04 and macOS 14 with Python 3.11.

## Key Interactions to Verify

- Run tests before any build in a fresh checkout; results must not depend on `.build` or `output`.
- Build two campaigns in both orders and concurrently; artifacts and reports remain isolated.
- Invoke the CLI outside the repository root; declared campaign and asset paths resolve correctly.
- Run `build-all --offline` without provider credentials; verify zero network/model calls.
- Change one allowed override and confirm only the declared resolved field changes.
- Submit unknown override and schema fields; confirm actionable failure rather than ignore.
- Remove Playwright/Chromium or force page load failure; verification must exit non-zero.
- Remove provenance, forge an unknown fact ID, use a missing evidence file, and attach an incompatible fact/template pair; all must block.
- Force copy/image provider timeout, rate limit, malformed response, and server error; no placeholder or empty content may be emitted as success.

## Edge Cases

- Empty YAML, YAML root of the wrong type, duplicate logical IDs, and cyclic/broken references.
- Absolute paths, `..` traversal, symlinks, spaces, Chinese filenames, and execution outside repo root.
- Two outputs with the same channel/type filename and two concurrent runs of the same campaign.
- Missing message plan, empty output target list, unknown template, unsupported ratio, and missing product asset.
- Objective claim disguised as subjective copy; fact ID exists but is unrelated to the selected factual template.
- Unicode units, decimals, ranges, percentages, comparative language, certifications, rankings, and absolutes.
- Browser installed but launch fails; generated HTML loads a missing local asset; screenshot is empty.
- Existing historical output from another brand or campaign.

## Critical Paths

- Fresh checkout -> install -> tests -> offline build -> verify -> inspect artifacts.
- Valid specs -> compile twice -> byte-identical declared artifacts and canonical manifest.
- Two brands -> four campaigns -> no Python changes -> isolated reports.
- Online generation -> validated structured response -> evidence-bound output -> explicit verification.
- Five modification tasks -> exact affected artifacts -> unchanged non-target hashes -> permitted model-call count.

## Required Test Layers

- Unit: schema rules, override policy, path resolution, diagnostics, claim/template binding.
- Integration: isolated compile/render/verify with temporary directories and fake providers.
- CLI: exit codes, flag forwarding, execution from another directory, `build-all` aggregation.
- End-to-end offline: clean checkout on both supported operating systems.
- Contract: recorded SenseNova response fixtures plus a minimal live smoke test before release.
- Manual: second-brand setup and non-author usability rubric.

## Regression Cases From This Review

- Tests must pass before a build creates `.build/resolved-task.json`.
- `build-all --offline` must not translate the flag into an unrecognized nested option.
- Visual compilation must retain `layout`, `product_image`, and `scene_policy`.
- Verification must not scan historical campaigns using the current campaign's resolved task.
- Missing Playwright or a browser exception must increment failures and block.
- Missing/forged provenance and nonexistent evidence references must block.
- Background API failure must not silently become an SVG placeholder in online mode.
- `build-all` must return non-zero if any child build fails and must not print unconditional success.
